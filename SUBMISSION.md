# 提交材料：价序

初赛材料匿名。赛道：医药价格治理。

## 项目名称

价序

## 一句话

价序把医保价格治理岗每天要核、要催、要回访的价格线索排成今日处置顺序。

## 项目摘要

医保价格治理岗每天早上面对一堆没排好顺序的线索。昨夜新增的机构执行价、集采采购进度、目录别名、投诉线索和昨日未回访事项常常混在一起。工作人员要先判断今天哪几件最该办，哪些要补证，哪些要派核验，哪些只能观察。

价序把这个早上动作做成工作台。用户打开 `/morning`，改一句今天的关注点，点 `开晨会`。系统读取合成/脱敏的价格来源、重点对象和未完待办，调用 live provider 规划今日优先级，再由服务端写入 `morning_session`、`daily_lead`、`follow_up_task`、`agent_run` 和 `replay_timeline`。

成功页显示今天先办哪些线索、为什么排在前面、还缺什么证据、下一步由谁处理。用户打开第一条线索后，可以选择 `退回补证` 或 `派核验`，线索状态和待办会写入数据库，回放记录来源、排序理由和人工边界。发函、通报、违规认定和关闭线索仍由业务人员确认。

项目覆盖医药价格数据标化治理、药品及医用耗材价格动态监测、价格异常预警与处置、集采价格落地跟踪。CSV 采集箱保留为补来源入口，不作为产品首页。

## 对照评审维度

### 场景契合度

首屏就是价格治理岗的早上工作：开晨会，排今日线索，处理第一条。字段、状态和待办围绕价格目录、中选价、执行价、集采落地、证据缺口和回访。

### 技术可行性

关键路径调用 OpenAI 兼容 live provider。服务端负责状态写入和人审边界。SQLite 保存晨会、线索、待办、运行记录和回放。Provider 失败时写失败态，不生成假线索。

### 产品完整度

评委可以完成一条完整路径：打开 `/morning`，点 `开晨会`，进入今日线索，打开第一条，退回补证或派核验，再看回放。队列、采集箱、设置页也能继续承接工作。

### 创新性

价序没有把价格治理做成大屏或审计表。它让智能体先承担“今天先处理谁、为什么、缺什么证据”的早上排序工作，再把下一步交回给人。

### 应用潜力

后续可接省级招采平台、机构执行价回传、采购订单进度、投诉回函和工单系统。当前演示不声称这些生产系统已接入。

## 演示路径

2. 修改“今天先看什么”。

4. 打开第一条线索。
5. 点 `退回补证` 或 `派核验`。
6. 查看待办和回放。
