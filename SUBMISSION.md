# 提交材料：医保可信数据通行 Agent

> 初赛材料须匿名，本文件不含团队、个人或机构标识。赛道：数据赋能。

## 项目名称

医保可信数据通行 Agent

## 一句话

弄脏一行医保数据，看 Agent 决定这批数据能否通行。

## 项目摘要

医保数据在进入基金监管分析、价格治理、经办服务推送和可信数据空间共享之前，一行错误编码、未来日期、身份冲突或越权访问，都可能污染整批数据。现有工具大多停在数据质量评分、规则命中列表和血缘报告，能解释风险，却不替运营人员推进状态。

本项目把「数据质量检查」改成「数据发布通行动作」。运营人员打开一个待发布批次，选中或改坏一行脱敏数据，点 `通行检查`。Agent 按 observe、plan、tools、mutate、verify、recover 运行：先用 live 大模型生成计划，再由服务端确定性工具校验 schema、医保目录字典、tokenized 身份、访问策略和日期异常，最后按安全优先级把这批 release 写到下一个状态，并落库行级问题、纠错提案、隔离项、审批任务、运行记录和回放时间线。

四种变更走出不同路径：错误编码命中高置信别名进 `纠错候选`，未来日期进 `隔离`，身份模糊匹配和越权访问进 `需审批`。大模型只负责规划，裁决和写库交给工具，敏感场景一律停到人审或隔离，不自动放行。provider 不可用时落 `检查失败`，不把规则结果包装成成功。

演示数据全部为合成脱敏 fixtures，身份用掩码 token，保留与真实可信数据空间一致的行结构和策略快照，可平滑替换为真实适配器后进入集中部署。评委 60 秒内能完成「改坏一行 → 状态改变 → 回放取证」的闭环。

## 对照评审维度

### 场景契合度（25%）

切口是数据赋能赛道里一个能操作的业务对象：dataset release。运营人员在数据出域、入湖、共享、建模取数前做通行把关。一行坏数据不再污染整批，而是被写到更安全的状态，并留下可回放证据。相邻场景覆盖价格治理的编码/单位/别名标准化、经办服务的日期与身份核验、基金监管的取数防污染。

### 技术可行性（25%）

- live 大模型在 plan 关键路径上，带 token 计量和真实网络延迟，证据见 `.hunter/live-provider-smoke.json`。
- 两个必需 live smoke 断言不同输入产生不同计划、工具、审批边界、状态和输出哈希，11 项断言全过。
- 全部业务对象用 `node:sqlite` 落库：release、version、row issue、correction、quarantine、approval、agent run、replay timeline。
- provider/存储/规则源故障显示降级，不伪装成功，证据见 `.hunter/degraded-run.json`。

### 产品完整度（20%）

新用户不用搭建即可操作 seeded release。七条路由覆盖通行检查、结果、回放、源清单与策略、审批、队列、设置。角色权限、生命周期、审批边界、回放、恢复路径都在。桌面和移动均可用，截图见 `docs/screenshots/`。

### 创新性（15%）

展示的是会改状态的 Agent 加可信数据空间原语，而不是又一个监测大屏或血缘报告。证明、日志、规则、回放都在状态变化后才披露，首屏只有一行数据、一个后果预览、一个主动作。

### 商业潜力（15%）

月 1 运营人员每周回到 release 队列处理新批次、隔离项和审批项；月 6 团队用多 release 工作台处理标准化、访问策略、审批、回放和审计。落点在医保数据中心和可信数据空间的日常运营。

## 演示路径

1. 打开 `/release/REL-2026-0623-07`。
2. 选一行，选变更类型，看后果预览。
3. 点 `通行检查`。
4. 状态从 `待发布` 变为 `纠错候选` / `隔离` / `需审批`。
5. 进 `运行回放` 看计划、工具调用、状态写入和 before/after。

## 数据与合规

合成脱敏 fixtures，掩码 token，界面标注 `合成脱敏数据`，无真实敏感医保数据。不声称已接入真实可信数据空间，仅保留迁移路径。不自动批准敏感发布。

## 六个月路线

接真实可信数据空间适配器，加价格治理、经办服务、基金监管专用模板，做批量 release 扫描、定时策略重放、跨数据集影响预览，以及多团队 workspace、SLA、通知和工单集成。

## 运行方式

见 `README.md`。核心命令：`npm run creds && npm install && npm run build && npm run start`，验证用 `npm run smoke:agent`。
